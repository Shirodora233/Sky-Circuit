# Sky Circuit 长期开发计划

> Status: historical direction draft. It is still useful for high-level design
> intent, but the immediate next steps and versioned scene references are
> outdated. Use the root `README.md` for current entry points, active scenes,
> build steps, and gameplay rules.

## 1. 项目定位

Sky Circuit 是一个原创 3D 空中竞技游戏，核心灵感来自文档中整理的 Flying Circus 规则，但不直接使用原作标题、角色、Logo、专有包装或美术素材。

目标不是先做完整商业游戏，而是逐步验证空中竞技是否好玩：先把“飞行、浮标得分、背部触碰、反弹、路线博弈”做成立，再扩展角色类型、AI、视觉表现、赛制和内容。

核心体验一句话：

```text
玩家在水面上方的立体赛道中高速飞行，通过抢浮标、切线拦截、背部触碰和反重力弹开，在限时比赛中取得更高分。
```

## 2. 必须保留的玩法骨架

从原设定中应优先保留这些机制：

- 4 个浮标组成空中赛道。
- 1v1 限时比赛。
- 触碰浮标得分，触碰对手背部得更高分。
- Dogfight 的目标是抢背后位置，不是击倒或扣血。
- 背部触碰后双方被弹开，弹开位置影响后续路线。
- Speeder / Fighter / All-Rounder 形成明显打法差异。
- Shortcut 让落后者或 Fighter 能放弃当前浮标去抢战术位置。
- Second 抽象为雷达、方向提示、危险提醒和战术 UI。
- 高度既是视野和俯冲加速资源，也会拉长路线。

## 3. 长期技术原则

前期实现要小，但架构要给后期留空间：

- 飞行控制、得分规则、角色参数、比赛流程、UI、AI 分层实现。
- 所有核心数值使用 ScriptableObject 配置，例如选手类型、飞行参数、得分规则、比赛时长、浮标布局。
- 先做单机 1v1，不在第一阶段碰联网。
- 先做可调试的灰盒玩法，不急于上高质量角色和特效。
- 用事件驱动连接系统，例如 `OnBuoyScored`、`OnBackHit`、`OnMatchEnded`，避免 UI 和规则直接耦合。
- 先保留 Arcade 手感，等核心循环成立后再逐步引入更真实的惯性、高度收益和高级技巧。

## 4. 版本路线

### V0.1 飞行手感原型

目标：玩家能在 3D 空间中稳定飞行，并且飞行本身有速度感。

内容：

- 第三人称飞行控制器。
- 加速、减速、转向、上升、下降。
- 基础 Balancer 辅助，让相机和角色姿态稳定可控。
- 一个水面上方测试场景。
- 简单速度 UI 和调试参数面板。
- 尾迹占位特效。

验收标准：

- 玩家能围绕 4 个空间点飞行一圈。
- 低速、高速、转向、俯冲不会明显失控。
- 相机不晕、不遮挡、不频繁丢目标。

### V0.2 第一版可玩比赛

目标：完成最小可玩的 Sky Circuit。

内容：

- 4 浮标方形空中赛道。
- 1v1 比赛流程：开始、倒计时、比赛计时、结束、结算。
- 玩家触碰当前目标浮标得分。
- 简单 AI 对手沿浮标路线飞行。
- 分数 UI、时间 UI、当前目标浮标提示。
- 出界或落水重置。
- 一局暂定 3 分钟，触浮标 +1。

暂不做：

- 完整 Dogfight。
- Shortcut。
- 复杂 AI 战术。
- 多角色、换装、剧情。

验收标准：

- 玩家可以和 AI 完成一整局。
- 比赛有明确胜负。
- 即使没有近身战，也能感受到空中竞速和抢点节奏。

### V0.3 Dogfight 与反重力弹开

目标：让游戏从普通飞行竞速变成 Sky Circuit。

内容：

- 对手背部判定区域。
- 玩家进入对手背后攻击角度和距离时可触背得分。
- 触背 +2。
- 触背后双方按接触方向弹开。
- Point Field 占位特效。
- 开局第一个浮标前禁止接触得分。
- AI 能做基础规避和追逐。

验收标准：

- 玩家能主动放弃浮标去追背。
- 触背后的位置变化会自然影响下一次路线选择。
- Dogfight 得分强但有风险，不是无脑贴近。

### V0.4 角色类型与路线博弈

目标：建立 Speeder / Fighter / All-Rounder 的差异。

内容：

- 三套角色参数：
  - Speeder：最高速高、加速慢、转向弱、适合抢浮标。
  - Fighter：最高速低、加速快、转向强、适合近身。
  - All-Rounder：中间值，适合切换策略。
- Shortcut 规则第一版。
- 领先者限制第一版，避免 Speeder 滚雪球。
- 雷达 UI 显示对手方向、距离和高度差。
- AI 根据类型使用不同策略。

验收标准：

- 三种类型玩起来有明显区别。
- 落后方有反打机会。
- 比赛不再只是最短路径竞速。

### V0.5 观赛表现与反馈

目标：让玩家看得懂高速立体比赛。

内容：

- 更清晰的浮标、线路、目标方向指示。
- Contrail 尾迹颜色区分选手。
- 背部得分 Point Field 特效。
- 反弹冲击波和音效。
- Second 风格提示：背后危险、对手 Shortcut、可攻击窗口。
- 简单回放或精彩瞬间慢动作占位。

验收标准：

- 玩家能判断自己该去哪、对手在哪、刚才为何得分或失分。
- 旁观者也能大致看懂比赛局势。

### V0.6 基础技巧系统

目标：加入可操作深度，但仍保持可控。

优先技术：

- Low Yo-Yo：下降换取加速。
- High Yo-Yo：上升减速诱导对手前冲，再俯冲反打。
- Scissors：左右急转扰乱追击。
- Airkick Turn：消耗能量或冷却的急转。

实现方式：

- 不一开始做复杂招式输入。
- 先把技巧拆成飞行物理和辅助控制参数。
- 部分高级动作可作为技能按钮或条件触发动作。

验收标准：

- 技巧能改变局势。
- 新手不用技巧也能完成比赛。
- 高手使用技巧会更有效率。

### V0.7 内容化与单人流程

目标：从原型变成可持续游玩的游戏。

内容：

- 训练模式。
- 单场比赛模式。
- 3 至 5 个 AI 对手。
- 简单杯赛或联赛结构。
- 不同浮标布局和天气时间。
- 角色/鞋参数解锁或配置。
- 设置菜单和按键重绑定。

验收标准：

- 玩家可以连续玩 20 到 30 分钟而不只是测试一局。
- 参数成长不破坏竞技公平性。

### V1.0 垂直切片

目标：做出可以展示给别人玩的完整小版本。

内容：

- 完整主菜单、训练、单场、杯赛。
- 3 个角色类型，至少 6 套参数化 AI。
- 2 到 3 张场地。
- 完整比赛 HUD。
- 基础音效和音乐。
- 可保存设置与成绩。
- 性能和手感打磨。

验收标准：

- 没有开发者解释，玩家也能完成教程并理解胜负。
- 核心循环足以支撑反复游玩。
- 美术即使不是最终品质，也有统一风格。

## 5. 第一版需要完成的内容

第一版建议定义为 V0.2，目标是“可完整打一局”，不要把 Dogfight 也塞进第一版。

第一版范围：

- 一个水面上方灰盒场地。
- 四个浮标和连接线。
- 玩家第三人称飞行。
- 一个基础 AI 对手。
- 当前目标浮标判定。
- 触浮标 +1。
- 3 分钟倒计时。
- 分数、时间、目标提示 UI。
- 比赛开始、结束、胜负结算。
- 出界或落水重置。

第一版暂缓：

- 触背得分。
- 反重力弹开。
- Shortcut。
- 三职业完整平衡。
- 高级技巧。
- 正式角色模型和动画。
- 联机。

第一版这么切的原因：

- 飞行手感和空间认知是最大风险，必须先验证。
- 如果一开始同时做 Dogfight、Shortcut 和复杂 AI，很难判断不好玩到底是控制问题、规则问题还是 AI 问题。
- V0.2 的代码结构仍要预留 `ScoreRule`、`CompetitorProfile`、`MatchState`、`FlightController`、`BuoyRoute`，后面加 Dogfight 不需要推倒重来。

## 6. 推荐系统拆分

### Gameplay

- `FlightController`：玩家/AI 共用的飞行运动接口。
- `PlayerFlightInput`：读取输入并转换为飞行指令。
- `AIPilotController`：AI 飞行指令生成。
- `Competitor`：选手状态、队伍、分数相关身份。
- `CompetitorProfile`：Speeder / Fighter / All-Rounder 参数配置。

### Match

- `MatchManager`：比赛状态机。
- `ScoreManager`：得分规则与事件。
- `BuoyRoute`：浮标顺序、当前目标、路线方向。
- `BoundaryVolume`：出界和落水检测。
- `RespawnSystem`：重置位置和短暂无敌。

### Combat

- `BackHitSensor`：背部得分判定。
- `MembraneRepulsion`：接触反弹。
- `DogfightAssist`：锁定、接近提示和攻击窗口。

### UI

- `MatchHUD`：时间、分数、当前目标。
- `RadarHUD`：对手方向、距离、高度差。
- `SecondHintUI`：文字或语音式提示。

### Presentation

- `ContrailRenderer`：尾迹。
- `PointFieldEffect`：背部得分反馈。
- `BuoyVisual`：浮标、线路和目标状态。
- `CameraRig`：第三人称飞行相机。

## 7. 外部资源需求

第一版可尽量少依赖外部资源，用 Unity 基础几何体和 URP 材质即可。

必要资源：

- Unity 6000.4.0f1，当前项目已使用。
- URP，当前项目已安装。
- Input System，当前项目已安装。
- 基础字体，使用 Unity/TMP 或系统默认占位字体即可。

建议尽早加入：

- Cinemachine：更快搭建第三人称飞行相机、目标跟随和镜头阻尼。
- TextMeshPro：如果项目模板未完整导入 TMP Essentials，用于正式 UI 文本。
- 一个简单水面 Shader 或水面资源，用于场地参照和落水反馈。
- 基础 SFX：加速、浮标得分、倒计时、反弹、警告。
- 占位人形或飞行器模型：可先用 Capsule，之后换原创角色。
- 粒子/VFX Graph 或简单 Trail Renderer，用于 Contrail 和 Point Field。

后期可能需要：

- 原创角色模型、飞行服、飞行鞋。
- 原创 UI 图标和品牌视觉。
- 比赛场地资产：海面、平台、观众席、浮标、灯带。
- 音乐与环境音。
- 更成熟的 AI 行为树或状态机工具。
- 如果做联网，再考虑 Netcode for GameObjects、Photon Fusion 或其他方案，但不应进入第一阶段。

版权注意：

- 不使用《苍之彼方的四重奏》的角色、Logo、名称包装、原作术语视觉设计或美术素材。
- “Flying Circus / FC / Skywalker / Grav-Shoes”等术语在对外版本中应替换为原创命名。
- Sky Circuit 可以保留“空中竞技、反重力鞋、浮标、背部触碰、尾迹、场外导航”这些机制层概念。

## 8. 最近三步

1. 建立项目目录和脚本命名规范。
2. 完成 V0.1 飞行控制器、相机和测试场。
3. 在 V0.1 手感可接受后，再进入 V0.2 的浮标比赛流程。

建议不要先做角色、美术或高级技巧。这个项目真正的地基是飞行手感、空间指引和得分规则。
